# Урок 21: Использование ИИ для генерации кода

## Основные темы

### Инструменты с ИИ
- **ChatGPT** - веб-интерфейс для генерации кода
- **GitHub Copilot** - интеграция в VS Code
- **Claude** - альтернатива ChatGPT
- **Codeium** - бесплатный помощник для кода

### Формулирование запросов
```
Хороший запрос:
"Создай Flutter StatefulWidget для экрана входа с двумя TextFormField
(email и пароль), валидацией и кнопкой входа"

Плохой запрос:
"Сделай форму"
```

### Принципы хороших промптов
1. Указывать язык/фреймворк (Flutter, Dart)
2. Описывать структуру (StatefulWidget, Provider)
3. Перечислять компоненты (поля, кнопки)
4. Указывать поведение (валидация, навигация)

### Пример: Запрос экрана входа
```
Создай Flutter виджет LoginScreen:
- StatefulWidget
- Два поля: email и пароль (TextFormField)
- Валидация: email должен содержать @, пароль минимум 6 символов
- Кнопка "Войти" с проверкой формы
- При успешной валидации показать SnackBar
```

### Работа с сгенерированным кодом
1. **Скопировать** код в проект
2. **Проверить** компиляцию
3. **Исправить** ошибки если есть
4. **Протестировать** функциональность
5. **Улучшить** код:
   - Переименовать переменные
   - Добавить комментарии
   - Улучшить UI
   - Добавить обработку ошибок

### Пример улучшения кода от ИИ
```dart
// До (от ИИ):
class LoginScreen extends StatefulWidget {
  @override
  _LoginScreenState createState() => _LoginScreenState();
}

// После (улучшенный):
class LoginScreen extends StatefulWidget {
  const LoginScreen({Key? key}) : super(key: key);

  @override
  State<LoginScreen> createState() => _LoginScreenState();
}
```

### Интеграция кода
```dart
// Добавление в существующий проект
import 'screens/login_screen.dart';

MaterialApp(
  home: LoginScreen(),
  routes: {
    '/login': (context) => LoginScreen(),
    '/home': (context) => HomeScreen(),
  },
)
```

### Типичные проблемы с ИИ-кодом
1. **Устаревший синтаксис** - обновить до текущей версии
2. **Отсутствующие импорты** - добавить нужные
3. **Hardcoded значения** - заменить на переменные
4. **Отсутствие null-safety** - добавить ? и !

### Чеклист проверки ИИ-кода
- [ ] Код компилируется без ошибок
- [ ] Все импорты на месте
- [ ] Переменные понятно названы
- [ ] Нет hardcoded строк (вынести в константы)
- [ ] Есть обработка ошибок
- [ ] UI выглядит аккуратно
- [ ] Код соответствует стилю проекта

## Ключевые концепции
- ИИ ускоряет разработку, но требует проверки
- Всегда тестируйте сгенерированный код
- Улучшайте код под свои стандарты
- Учитесь формулировать чёткие запросы

## Ограничения ИИ
- Может использовать устаревшие API
- Не знает контекст вашего проекта
- Может генерировать неоптимальный код
- Требует проверки безопасности
